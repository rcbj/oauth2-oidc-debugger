OIDC Dynamic Client Registration
 

Create, read, update, and delete a client on the Identity Provider using the OIDC Dynamic Client Registration protocol. See the OIDC Registration spec (and RFC 7591 / RFC 7592).
The OIDC Dynamic Client Registration Endpoint. Kept in sync with the Registration Endpoint in the Configuration Parameters pane above, and auto-populated from the discovery metadata (registration_endpoint).
Optional bearer token presented when creating a client. Some Identity Providers require an initial access token to register; leave blank for open registration.
The client metadata sent on create (POST) and update (PUT). Populated from the discovery metadata and the Redirect URL above.
The HTTP request that "Register New Client" will send to the Registration Endpoint (a POST of the client metadata).
 
The client configuration endpoint (registration_client_uri) returned at registration. Used to read, update, and delete the client. Editable so you can manage a previously registered client.
The registration_access_token returned at registration. Presented as a bearer token to read, update, and delete the client (RFC 7592).
The response from the Dynamic Client Registration endpoint.
